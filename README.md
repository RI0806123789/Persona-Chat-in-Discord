# Persona Chat in Discord

## 🛠️ 技術スタック

![Python](https://img.shields.io/badge/Python-3.10%2B-3776AB?style=for-the-badge&logo=python&logoColor=white)

Google Gemini AI を活用した Discord ボットです。
テキストチャット・ボイスチャット読み上げ・ペルソナ切り替え・利用状況グラフ表示などの機能を備えています。

> **⚠️ フィクションに関する免責事項**
>
> 本プロジェクトに含まれるペルソナ（キャラクター設定）は、すべて**架空のフィクションキャラクター**です。
> 各プロンプトファイルに記載された性格・口調・行動パターンは、エンターテインメント目的のロールプレイ指示であり、
> 実在の人物・団体・属性を表現・差別・攻撃する意図は一切ありません。
> 本ボットの出力はすべて AI による創作であり、開発者の個人的な見解や主張を反映するものではありません。

## 🛠️ セットアップ

### 1. 事前準備

以下を用意してください。

- **Python 3.10 以上**
- **Discord Bot トークン**（[Discord Developer Portal](https://discord.com/developers/applications) で取得）
- **Gemini API キー**（[Google AI Studio](https://aistudio.google.com/app/apikey) で取得）
- **FFmpeg**（後述）

---

### 2. パッケージのインストール

```bash
pip install discord.py google-generativeai gTTS PyNaCl httpx matplotlib numpy Pillow PyPDF2
```

または `requirements.txt` を使用:

```bash
pip install -r requirements.txt
```

---

### 3. FFmpeg のセットアップ

ボイスチャンネルでの読み上げに FFmpeg が必要です。

> ⚠️ FFmpeg のバイナリ・DLL はサイズが大きいため Git 管理対象外です。必ず手動でダウンロードしてください。

1. [ffmpeg.org/download.html](https://ffmpeg.org/download.html) から Windows 用ビルド（shared版）をダウンロード
2. `ffmpeg.exe` / `ffplay.exe` / `ffprobe.exe` と各 `.dll` ファイルを `index/` フォルダ（`bot.py` と**同じフォルダ**）に配置

---

### 4. APIキーの設定

`index/config.py` に、以下の内容を記述してください:

```python
API_KEY_GEMINI = "YOUR_GEMINI_API_KEY"
TOKEN_DISCORD  = "YOUR_DISCORD_BOT_TOKEN"
```

---

### 5. ボットの起動

```bash
cd index
python bot.py
```

`ログイン成功: BotName#0000` と表示されれば起動完了です。

## 💬 コマンド一覧

### `/functions`

各種機能をまとめた「機能パネル」を開きます。パネルは**本人だけに見える（エフェメラル）**状態で表示され、ボタンで各機能に切り替わります。各パネルは120秒で自動的にタイムアウトします。

> Discord の UI は最大5行までのため1枚に合体できません。選んだ機能のパネルへ**同じメッセージを差し替える**方式を採用しています。各パネルの「**↩️ 戻る**」ボタンで機能パネルへ戻れます。

| ボタン            | 機能                                                                          |
| ----------------- | ----------------------------------------------------------------------------- |
| ⚙️ キャラ設定     | キャラ設定パネルに切り替え（ペルソナ・モデル・応答モード・会話リセット）       |
| 📋 会話要約       | 会話要約パネルに切り替え（期間・表示範囲を選んで議事録を生成）                 |
| 📈 会話統計       | 会話統計パネルに切り替え（発言回数・時間帯・話題・性格推定をグラフ表示）       |
| 💬 自発会話       | 自発会話パネルに切り替え（自発的な話題提供の ON/OFF・無会話しきい値を設定）   |
| 📊 グラフ         | 利用トークン数の推移グラフをその場で表示                                      |
| 🔄 NGワード再読込 | NGワードリストを再起動なしで再読み込み                                        |
| 🔍 Google検索     | Google検索グラウンディングの ON/OFF をその場で切り替え（デフォルト: ON）      |
| 🚫 ブロック解除   | ブロック中ユーザーの選択メニューから解除（**管理者のみ表示**）                |

#### ⚙️ キャラ設定

**操作方法：**

1. 「⚙️ キャラ設定」を押すと、現在の設定と3つのセレクトメニュー・各種ボタンが表示されます
2. セレクトメニューから変更したい項目を選択します（複数の項目を変更可能。選択結果はメニュー内に即反映されます）
3. 「**確定**」ボタンを押すと設定が反映されます
4. 「**↩️ 戻る**」ボタンを押すと変更は破棄され、機能パネルに戻ります
5. 「**🔄 会話リセット**」ボタンを押すと、そのチャンネルの会話履歴がリセットされます

※ Google検索グラウンディングの ON/OFF は機能パネルの「🔍 Google検索」ボタンで切り替えます。

**セレクトメニュー：**

| メニュー      | 選択肢                                                                                                   |
| ------------- | -------------------------------------------------------------------------------------------------------- |
| 👤 ペルソナ   | ツンデレ / ヤンデレ / メイド / ロリ / ショタ / オジサン / ギャル / メスガキ / お姉さん / はんなり / クレーマー / 標準 |
| 🤖 AIモデル   | Gemini 2.5 Flash / Gemini 3.5 Flash / Gemini 3.1 Flash Lite                                              |
| 💬 応答モード | すべての発言に反応 / メンションのみに反応（デフォルト）                                                  |

#### 📋 会話要約

そのチャンネルの会話を、発言者名つきの議事録として要約します。

1. 「📋 会話要約」を押すと、期間ドロップダウン・表示範囲トグル・「要約実行」ボタンが表示されます
2. **期間**（1時間 / 12時間 / 1日 / 1週間 / 1か月）を選択します
3. **表示範囲**（🌐 全員に表示 / 🔒 自分だけ）をトグルで切り替えます
4. 「**要約実行**」で議事録を生成します（🌐 のときはチャンネルに投稿、🔒 のときは本人だけに表示）

※ 要約はキャラ設定パネルで選択中のモデル・ペルソナを使用します。対象メッセージが 1000 件を超える場合は自動停止します。

#### 📈 会話統計

ユーザーごとの会話傾向を、**本人だけに見える（エフェメラル）**ダッシュボードで表示します。集計対象は「ボットへの発言のみ」です。

1. 「📈 会話統計」を押すと、項目セレクト（全部 / 時間帯 / 話題 / 性格推定）とグラフが表示されます
2. 表示できる内容は次の通りです
   - **時間帯**: よく話す時間帯の発言回数ヒストグラム（`Asia/Tokyo` 基準の 24 時間）
   - **話題**: 直近の発言から推定した「よく話す話題」の重み付き横棒グラフ
   - **性格推定**: 直近の発言から推定したビッグファイブ性格特性のレーダーチャート

※ 話題・性格は Gemini による推定で、結果は 24 時間キャッシュされます。性格推定は発言が 10 件未満のときは「データ不足」として表示されません。
※ 一般ユーザーは**自分の統計のみ**閲覧できます。**管理者のみ**、他メンバーの統計や発言回数ランキング（全体比較）も閲覧できます。

#### 💬 自発会話

一定時間会話が無いチャンネルへ、ボットの方から過去の話題をもとに話しかける機能です（デフォルト: OFF）。

1. 「💬 自発会話」を押すと、しきい値ドロップダウンと ON/OFF トグル、確定/戻るボタンが表示されます
2. **無会話しきい値**（1時間 / 3時間 / 6時間 / 12時間 / 24時間）を選択します
3. 「**💬 自発的な会話**」トグルで機能の ON/OFF を切り替えます
4. 「**確定**」ボタンを押すと設定が反映されます（押すまで設定は変更されません）

※ 対象は「ボットが会話に参加したことのあるテキストチャンネル」で、自動的に監視対象へ登録されます（DM は対象外）。
※ ボット宛てでない発言もそのチャンネルの「会話」としてカウントされ、会話が活発な間は割り込みません。
※ 一度話題提供したら、誰かが発言するまで同じチャンネルには再投稿しません（沈黙中の連投防止）。
※ 話題はキャラ設定パネルで選択中のペルソナ・モデルと、チャンネル履歴・長期記憶（プロフィール）をもとに生成されます。
※ 応答モードが「メンションのみに反応」の場合、話題提供への返事にもメンションが必要です。
※ 設定・監視状態はボット再起動でリセットされます（再起動後はデフォルトの OFF に戻ります）。

#### 📊 グラフ / 🔄 NGワード再読込

- **📊 グラフ**: 入力・出力・合計トークン数の推移をグラフ画像で表示します（ボット再起動でデータはリセット）
- **🔄 NGワード再読込**: `ng/NG_WORD.txt` を再起動なしで再読み込みします

#### 🚫 ブロック解除（管理者専用）

三審制ブラックリストでブロックされたユーザーを、`ng/blacklist.json` を直接編集することなく解除し、違反カウントをリセットします。ブロック中ユーザーの選択メニューから対象を選びます。管理者以外にはこのボタンは表示されません。

## 📋 更新履歴

### ver0.1.4 — 2026/07/16

#### 永続化データを SQLite データベースへ移行

`memory/` 配下の構造化データ（`affinity.json` / `user_stats.json` / `queue.json` / `pending_log.txt`）と プロフィール記憶（`profile.json`）を、単一の SQLite データベース `memory/bot.db` に統合しました。SQL 文は新設の `database.py` に集約しています。`projects.json` は LLM が構造を自由に決める記憶のため、従来どおり JSON ファイルのまま扱います。

- **旧データの自動移行**: 初回起動時に旧 JSON / テキストファイルを DB へ取り込み、`<元の名前>.imported.bak` に改名して保全（二重取り込み防止＋バックアップ）
- **profile.json は手書き追加用の受け皿に**: 起動のたびに `profile.json` の中身を DB の `profile` テーブルへ自動マージ（同名キーはファイル側が優先）し、ファイルを `{}` に戻す。**ユーザーが `profile.json` に事実を書き足して再起動するだけで、ボットの記憶へ追加される**
- **スキーマ移行機構**: `PRAGMA user_version` によるバージョン管理を導入。将来列を追加したくなったら `SCHEMA_VERSION` を上げて `_MIGRATIONS` に `ALTER TABLE` 文を1行追記するだけで、既存 DB にも自動適用される
- **クラッシュ耐性の向上**: 書き込みがトランザクションで原子的になり、旧方式の「ファイル全体上書き中のクラッシュでデータ破損」リスクを解消
- **WAL モード**: ボット稼働中でも `sqlite3` CLI などから安全に中身を読める
- **文書キューの堅牢化**: タスクの取得（claim）を SELECT + UPDATE の同一トランザクションで行い、二重取得を構造的に防止
- 記憶圧縮のしきい値を「行数」から「Q/A 件数」（`MEMORY_FLUSH_THRESHOLD_ENTRIES`）に変更

#### 全機能を `/functions` パネルに統合（UI刷新）

`/status`・`/graph`・`/ng_reload`・`/unblock` と後述の会話要約を、単一の `/functions` コマンド（機能ハブ）に統合しました。Discord の View は最大5行までのため1枚のパネルに合体できないので、ボタンで選んだ機能のパネルへ同じメッセージを差し替える方式を採用しています。

- `/functions` を実行すると、本人だけに見える（エフェメラル）ハブパネルが開く
- **⚙️ キャラ設定** / **📋 会話要約** … パネルを開く（同じメッセージが変身）
- **📊 グラフ** / **🔄 NGワード再読込** … その場で実行し、別のエフェメラルメッセージで結果を返す
- **🚫 ブロック解除**（旧 `/unblock`・**管理者専用**） … `ng/blacklist.json` を直接編集せず、ブロック中ユーザーの選択メニューから解除し違反カウントもリセット。管理者以外にはボタン自体を非表示（従来は権限が無い場合に警告を返信していた）

#### チャンネル会話の議事録要約機能を追加

- **DM からの文書添付が必ず失敗する問題を修正** (`document_service.py`): 文書ワーカーのチャンネル種別チェックに DM チャンネルが含まれておらず、DM で受け付けた文書タスクが処理段階で必ずエラーになっていた。`on_message` 側と同じ判定（`is_supported_message_channel`）に統一。

不適切な発言を繰り返すユーザーを自動でブロックする三審制のブラックリスト機能を追加しました。
ユーザーの識別にはDiscordの固有ID（スノーフレークID）を使用しており、ユーザー名を変更しても回避できません。
違反データは `ng/blacklist.json` に永続化され、Bot再起動後も維持されます。

- ブロック解除は `ng/blacklist.json` を直接編集して対応

#### 感情・親密度パラメータ（2次元モデル）の導入(BETA)

ユーザーごとの長期的な関係性を示す「親密度」と、短期的な感情変動を示す「機嫌・テンション」のパラメータを導入し、より人間らしい自然な会話の文脈維持が可能になりました。
パラメータは `memory/affinity.json` にスレッドセーフで永続化されます。

- **長期関係（親密度 - Affinity）**: 0〜100（初期値50）。ユーザーの会話が好意的か攻撃的かによって緩やかに変動します。
- **短期感情（機嫌 - Valence / テンション - Arousal）**: それぞれ 0〜100（初期値50）の2次元モデルでその時の感情状態を表現します。
- **時間経過による感情リセット**: 会話のない経過時間に応じて、感情パラメータ（機嫌・テンション）は半減期（30分）モデルで自動的に初期値（50）へ減衰・リセットされます。
- **軽量・高速なタグ方式**: AIに通常のテキストとして返答させ、末尾に `[V:X, A:Y]` のタグを出力させることで、遅延のない最速のレスポンス設計を採用しています（タグはユーザーには非表示）。

- **デフォルト応答モードの変更**: デフォルトの応答モードを「すべての発言に反応」から「メンションのみに反応」に変更
- **DM メンションチェックの修正**: メンションモード時に DM チャンネルでの会話が無視されるバグを修正。DM ではメンションチェックをスキップするように変更
- **ニックネームメンション除去の修正**: `<@!id>` 形式（ニックネームメンション）がテキストから除去されず、そのまま AI に送信されていたバグを正規表現で修正
- **DM チャンネル応答不可の修正**: `is_supported_message_channel` に `DMChannel` が含まれておらず、DM での応答が生成されなかったバグを修正
- **Bot応答の AI モデレーション追加**: 直接会話での Bot 応答に AI モデレーションチェックが行われていなかった問題を修正。文書処理と同等の安全チェックを通常チャットにも適用
- **モデレーション誤ブロックの修正**: NGワードリスト自体がプロンプトに含まれることで、Geminiの安全フィルタが過剰に発動し `PROHIBITED_CONTENT` で安全な返答もブロックされる問題を修正
- **記憶圧縮モデルの安定化**: 記憶整理時にユーザー選択モデルを使用していたものを固定のルーターモデルに変更し、安定性を向上
- **JSON抽出処理の安全性向上**: 正規表現を非貪欲マッチに変更し、Markdownのコードブロック除去を追加して安全にJSONをパースできるように修正
- **グラフ描画のフォント設定局所化**: matplotlibのフォント設定を `plt.rc_context` で局所化し、他のモジュールへの影響を防止
- **音声読み上げ（TTS）の非同期化**: 再生終了の待機ループを `after` コールバックに変更して非同期化し、一時ファイルの保存先をOSの一時フォルダに変更
- **複数ファイル添付時のバグ修正**: 複数の文書を添付した際にBotがファイルの数だけ回答を繰り返すバグを修正。要約抽出のみをループし、応答は一度だけ行うようリファクタリング

### ver0.1.2 — 2026/07/07

#### 長期記憶システムを追加

Discord ボットの記憶処理をカテゴリ分割・ルーティング型に刷新しました。
`profile.json`、`projects.json`、`pending_log.txt` を使って、必要な記憶だけを選択的に読み込み、返信後は非同期で一時ログへ追記します。

判定に失敗した場合は `none` として安全側に倒し、`pending_log.txt` が一定量を超えるか長時間アイドルのときにバックグラウンドで圧縮・統合します。

- 軽量モデルで `profile / projects / none` を判定し、必要な記憶だけを結合
- 回答後に会話ログを `pending_log.txt` へ非同期追記
- バックグラウンドで一時ログを圧縮し、`profile.json` と `projects.json` を更新
- 圧縮失敗時は JSON を上書きせず、`pending_log.txt` を保持してフェールセーフ動作

#### 文書添付の非同期受付と要約記憶を追加

ファイル添付メッセージを受付スレッドで即時キュー登録し、ワーカースレッドで文書の検証・要約・応答生成・クリーンアップまでを分離しました。長文ドキュメントはそのまま会話に流さず、Gemini で圧縮要約して `memory/projects.json` の長期記憶へ保存します。

- `attachment.size` による事前サイズ検証で、大きすぎるファイルを先に門前払い
- 対応拡張子は `pdf / txt / csv / md` に限定し、未対応ファイルを即座に拒否
- ワーカー側で文書を一時保存し、検証後に Gemini へアップロードして要約を抽出
- 圧縮済み要約を `projects.json` の `document_summaries` に保持し、最終回答のコンテキストに再利用
- `genai.delete_file()` とローカル一時ファイル削除で、処理後の残存物を確実に破棄

- 任意でNGワードを設定し、不適切な表現を削減することが出来ます。

#### 新キャラクターの追加

#### リファクタリング・バグ修正

### ver0.1.1 — 2026/06/29

#### コマンドいらずの会話

`/ai` コマンドなしで、チャンネルへの通常の発言に直接反応するようになりました。`/setting` コマンドで「すべての発言に反応」と「メンションのみに反応」の2つのモードを切り替えられます。

- **すべての発言に反応**（デフォルト）: チャンネルのメッセージすべてに返答
- **メンションのみに反応**: `@Bot` でメンションしたときだけ返答

#### マルチモーダル対応

メッセージに画像を添付すると、テキストと合わせて Gemini に送信され、画像の内容を踏まえた返答が得られます。

- テキストなしで画像のみ添付した場合は、自動的に「この画像について説明・反応してください。」というプロンプトで処理されます
- 複数枚の画像も一度に送信可能です
- 対応形式: JPEG・PNG などの一般的な画像形式（`image/*` 全般）

#### 会話リセット

`/reset` コマンドでチャンネルの会話履歴をリセットできます。
リセット後は、それ以前の発言を参照せずに新しい文脈で会話を始めます。
リセットポイントはチャンネルごとに独立しており、ボット再起動でクリアされます。
